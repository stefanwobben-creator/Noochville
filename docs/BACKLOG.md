# Backlog — user stories

Product-backlog van NoochVille, geordend per thema. Near-term tech-taken staan in
`STATE.md` onder "Backlog (na vrijdag)"; dit bestand is de bredere wensenlijst.

## Rolbeheer
- Als leadlink wil ik op een roldetailpagina de role fillers kunnen aanpassen en AI agents kunnen toevoegen
- Als gebruiker wil ik in de navigatie alleen de cirkels uitgeklapt zien waar ik rollen in vervul

## Strategie & overzicht
- Als gebruiker wil ik de strategie zien waar nu overview is (strategie tab kan dan weg)
- Als gebruiker wil ik een duidelijk leesbare strategiepagina met correct gebruik van headings, kolommen, lijstjes en witruimte

## Rollenpagina Nooch
- Als gebruiker wil ik op de rollenpagina van Nooch de cirkel NoochVille weergegeven zien als een rol (met cirkel-icoontje), met daaronder de purpose en als rolvervullers de rep link en leadlink van de subcirkel (dynamisch: aanpassing in de subcirkel verandert weergave op beide plekken)
- Als gebruiker wil ik van de Nooch-cirkel ook de relevante projecten van subcirkels zien

## Mobile
- Als mobile user wil ik dat het scherm mobile friendly is

## Uploads
- Als gebruiker wil ik een PDF kunnen uploaden bij een project
- Als gebruiker wil ik een PDF kunnen uploaden aan de Librarian, die dan zelf relevante inzichten eruit haalt

> Let op: mogelijk geblokkeerd door "PDF-upload bug B1" (zie STATE.md → Backlog na vrijdag).
> Het `attach_file`-uploadpad loopt bovendien buiten `dispatch` om en heeft nog geen autorisatie.

## Projecten
- Als gebruiker wil ik als ik een project toevoeg alleen de projecten zien die passen bij mijn rol, en bij trekker standaard al mijn naam zien
- Als gebruiker wil ik bij een project geschatte tijdsduur, punten of impact/effort kunnen toevoegen via een codeconventie (bijv. #ime:1uur #points:5)

## Metrics & KPIs
- Als gebruiker wil ik alleen KPIs kunnen selecteren die ook data bevatten
- Als leadlink wil ik in de metrieken-catalogus zien welke KPIs actuele data bevatten
- Als facilitator wil ik een eigen dashboard met werk- en roloverleg-metrics

## Deploy & samenwerking
- Als website developer wil ik gemakkelijk maar ook veilig naar de server kunnen deployen
- Als website developer wil ik een functionaliteit zodat mensen suggesties kunnen plaatsen die ik kan verwerken in een backlog die inzichtelijk is voor iedereen
  → uitgewerkt ontwerp: `docs/ONTWERP_backlog_builder.md` (bouw gepland).

## Noochie
- Als gebruiker van Noochie wil ik dat Noochie ook bij alle dingen kan die ik als gebruiker zie, LLM toepast en goede adviezen geeft zodat ik mijn werk beter kan uitvoeren
