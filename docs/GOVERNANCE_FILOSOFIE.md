# Purpose vs. accountability — de filosofie achter governance

> Stefan, 2026-06-25. Leidend principe voor hoe een kans door governance stroomt.

## De kern
Een **purpose** is de noordster: vrijheid + autoriteit om te handelen zolang je geen domein van een
ander schendt. Je mag elke kans najagen die de purpose dient. Een **accountability** is een belofte
aan de stam: een doorlopende activiteit waarvan anderen mogen verwachten dat jij er energie in steekt.

De governance-vraag bij een nieuwe kans: handelen we vanuit **vrij verlangen** (purpose) of vanuit
**structurele plicht** (accountability)?

## Drie lenzen
- **Aristoteles (teleologisch):** purpose = het *telos* (waartoe, verandert zelden);
  accountabilities = de *middelen*. Een nieuwe, onbewezen manier om het telos te dienen is een
  *uiting van de purpose*. Pas als die manier een permanent, noodzakelijk onderdeel moet worden,
  *stolt* hij tot accountability.
- **Sartre (existentieel):** existentie gaat vooraf aan essentie. Purpose = absolute vrijheid
  (creëren, experimenteren, falen); accountability = het contract waarmee je je vrijheid inperkt om
  betrouwbaar te zijn. Vraag: *vereist deze kans dat ik mijn vrijheid inperk om voorspelbaarheid
  voor anderen te creëren?* Structurele relatie → accountability. Eenmalige verkenning → purpose.
- **James/Dewey (pragmatisch):** kijk naar de praktische frictie. Kun je het nú zelf najagen
  zonder dat iemand je tegenhoudt? → purpose. Strandt het omdat anderen geen tijd/prioriteit
  maken? → pragmatische spanning → vertaal naar accountability zodat je formeel aandacht mag eisen.

## De vuistregel (leidend voor het dorp)
1. **Start bij de purpose (experimenteren):** pak de kans op als project/proefballon, vanuit
   individuele actievrijheid. Niet aan de voorkant dichttimmeren.
2. **Observeer de frictie:** kost het structureel energie? Wachten andere rollen op de uitkomst?
   Ontstaan onduidelijke verwachtingen?
3. **Veranker als accountability (formaliseren):** zodra de kans van "leuk experiment" naar
   "noodzakelijke schakel" gaat, breng je 'm in het roloverleg. De kans is volwassen geworden.

## Wat dit betekent voor de code
- De **default** van een kans is een **project (experiment)**, niet een accountability.
- Een **accountability is een stolling**: hij hoort *bewijs van terugkerende frictie* te dragen
  (spiegelt G0's herhalingseis voor `add_role`).
- De **purpose** wijzig je zelden, en alleen om de ziel/het telos scherper te maken.
- Bij twijfel: liever een project voorstellen dan een accountability minten.
